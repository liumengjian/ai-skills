# 二十、jjb-cmd 命令行工具使用指南

## 5. 发布云组件

**⚠️ 前置条件**：执行发布命令前，必须先执行 `jjb-cmd auth` 进行登录授权，否则无法执行发布命令。

### 5.1 命令格式

```bash
jjb-cmd publish <版本号>
```

**说明**：发布云组件到组件库。

**执行位置**：在组件根目录执行。

**参数说明**：
- `<版本号>`：发布版本，可设置为 `latest` 或具体版本号（如 `1.0.0`）

### 5.2 版本号说明

发版版本号：

- **指定版本号**：例如 `1.0.0`、`1.2.3` 等
- **latest**：设置为 `latest`（最新版本）

**示例**：

```bash
# 发布指定版本
jjb-cmd publish 1.0.0

# 发布为最新版本
jjb-cmd publish latest
```

### 5.3 发布失败排查

若发布失败，可能存在以下问题：

- **登录授权问题**：没有登录授权成功，请确认是否执行 `jjb-cmd auth` 命令
- **权限问题**：当前账号没有当前组件所属的分类权限，需要登录授权对应账号
  - 当前组件只有两种类型：
    - **系统组件**
    - **业务组件**
  - 账号联系开发者获取！
