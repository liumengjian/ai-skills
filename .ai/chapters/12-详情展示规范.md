# 十二、详情展示规范

本规范旨在统一系统内所有详情类场景（Modal、Drawer、独立页面）的交互体验与视觉呈现。

### 1. 加载态管理 (Loading State)

- **强制约束**：所有异步查询详情数据的场景 **必须** 具备 Loading 状态。

- **实现方式**：

  - **Modal / Drawer 详情**：使用组件自带的 `loading` 属性或骨架屏（Skeleton）。

  - **独立页面详情**：使用全局 Loading 或局部 Spin 覆盖，确保用户在数据加载期间获得明确的反馈。

### 2. 数据呈现规范 (Data Presentation)

- **结构化展示**：详情页中的**纯文本数据展示**，**必须** 使用 `antd/Descriptions` 组件进行结构化排版。

- **禁止行为**：

  - 禁止使用原生的 `div`、`p` 标签进行简单的文本堆砌。

  - 禁止使用 `Row` / `Col` 手动模拟表单布局，除非 `Descriptions` 无法满足极特殊的排版需求。

- **视觉一致性**：应根据数据量的大小，合理配置 `Descriptions` 的 `column` 属性（通常建议为 2 或 3）。

### 3. 数据溯源逻辑 (Data Sourcing)

**强约束：AI 禁止自主选择数据获取方式。必须严格根据开发者在提示语中的明确指令，采用以下指定方式之一：**

| 开发者指令关键词    | 数据来源规范                                                            |
| :------------------ | :---------------------------------------------------------------------- |
| **"使用行数据"**    | **上下文获取**：直接从表格行对象（Record）中提取数据注入详情。          |
| **"通过接口获取"**  | **异步查询**：根据传入的唯一标识符（如 `id`）调用详情接口获取完整数据。 |
| **"通过地址栏 ID"** | **路由溯源**：通过 `router.query` 获取地址栏 `id`，并据此调用接口查询。 |
