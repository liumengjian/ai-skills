# 十二、详情展示规范

本规范旨在统一系统内所有详情类场景（Modal、Drawer、独立页面）的交互体验与视觉呈现。

## 3. 数据溯源逻辑 (Data Sourcing)

**强约束：AI 禁止自主选择数据获取方式。必须严格根据开发者在提示语中的明确指令，采用以下指定方式之一：**

| 开发者指令关键词    | 数据来源规范                                                            |
| :------------------ | :---------------------------------------------------------------------- |
| **"使用行数据"**    | **上下文获取**：直接从表格行对象（Record）中提取数据注入详情。          |
| **"通过接口获取"**  | **异步查询**：根据传入的唯一标识符（如 `id`）调用详情接口获取完整数据。 |
| **"通过地址栏 ID"** | **路由溯源**：通过 `router.query` 获取地址栏 `id`，并据此调用接口查询。 |
