# 三、项目页面结构

## 6. 动态路由规范

- **ID 动态路由**：当页面需要支持动态路由参数（如详情页、编辑页）时，文件夹命名使用 `_id` 后缀

  - 示例：`_id`、`_type`

  - 说明：系统会自动将 `_id` 识别为动态路由参数，并在路由中注入对应的 `id` 值

- **目录结构示例**：

  ```
  List/
  ├── index.js          # 列表页
  └── _id/
      └── index.js      # 动态路由详情页
  ```

- **代码示例**：

  **列表页**（`List/index.js`）：

  ```javascript
  export default props => {
      return props.children;
  };
  ```

  **动态路由详情页**（`List/_id/index.js`）：

  ```javascript
  import React from 'react';

  const List = (props) => {
      const { params } = props.match;
      const { id } = params;
      return <div>List {id}</div>;
  };

  export default List;
  ```

  **说明**：在动态路由页面中，可以通过 `props.match.params` 获取路由参数，其中 `id` 对应 `_id` 文件夹中的动态参数值
